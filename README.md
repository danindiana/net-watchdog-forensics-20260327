# Net Watchdog Forensics

A suite of tools for NIC hardening, network noise suppression, and automated forensic analysis using local LLMs.

## Project Scope
This project was born out of a network triage session on March 27, 2026, to resolve latency and "poor" connection reports on an Intel X540-T2 NIC.

## Tools
### 1. `harden_nic.sh`
Optimizes the Intel ixgbe driver settings:
- Doubled Ring Buffers (8192)
- Interrupt Coalescing (96us)
- MTU 1500 Enforcement

### 2. `suppress_noise.sh`
Mitigates Layer 2 noise:
- Drops AT&T BGW320 Proprietary Heartbeats (Ethertype 0x7373) using `ebtables`.
- Blocks UPnP/mDNS bloat using `UFW`.

### 3. `analyze_state.py`
The **Ollama-Delegate** implementation:
- Gathers system process and socket state.
- Automatically filters kernel noise.
- Offloads complex forensic pattern matching to local Ollama models (default: `deepseek-r1:14b`).

## Smart VRAM Management
The forensic tools are designed for multi-tenant environments (e.g., Gemini CLI + Claude Frontier). 
- **Dynamic Selection:** `analyze_state.py` automatically queries `nvidia-smi` to find the GPU with the most free overhead and selects a model that fits comfortably.
- **Manual Override:** Users can force extremely lightweight models (e.g., `qwen3.5:0.8b`) when VRAM is critically low.

## Multi-Agent Coordination
This project implements a shared `COORD.md` protocol to prevent conflicting system changes when multiple AI agents are active.
- **Shared Consensus:** Both Gemini and Claude have verified the system as "Safe to Reboot" with Ethernet as the primary path.
- **Locking:** Protocol includes checking for active task locks before modifying persistent networking configurations (Netplan/NetworkManager).

## Verification Success
As of March 27, 2026, this toolkit has been successfully verified against both standard and advanced forensic scenarios:
- **Standard Detection:** Correctly identified userspace scripts targeting external IPs via ICMP.
- **Red-Team Masquerade Detection:** Successfully flagged a malicious binary masquerading as a kernel thread (`[kworker...]`) by detecting its non-standard possession of a network socket.
- **VRAM Efficiency:** Proven successful using the `qwen3.5:0.8b` model (~1GB VRAM), confirming that forensic analysis can "float" on top of other heavy GPU tasks without causing model-swapping latency.
